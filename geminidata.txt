**The core workflow:** Ingest a 9x9 game board screenshot from the device clipboard, parse the grid mathematically using lightweight Dart isolates, pass the flattened state across a C-ABI bridge, and rely on a Rust-based state machine to compute the exact next logical move without blocking the UI.

## 2. Architecture & Tech Stack Rationale
The application relies on a tri-tier architecture leveraging a pre-existing Flutter/Rust/Julia FFI template.

### 2.1 Flutter/Dart (The Presentation & Intake Layer)
* **Role:** UI rendering, clipboard management, and raw image pixel-sampling.
* **Why Dart here?** Keeps the UI running at 60/120fps. By offloading image math to a Dart Isolate, we avoid blocking the main thread while maintaining cross-platform compatibility without external CV libraries.

### 2.2 Rust (The Core Logic Engine)
* **Role:** The Constraint Satisfaction Problem (CSP) solver and state machine.
* **Why Rust?** * **Zero Garbage Collection (GC) Pauses:** When the solver hits a roadblock and relies on Depth-First Search (DFS) or bifurcation trees, it creates and destroys thousands of temporary board states. Dart or Go would trigger GC stutters. Rust runs at raw hardware speed.
  * **Memory Safety:** Deep recursive backtracking in C/C++ risks stack overflows; Rust guarantees safety.
  * **C-ABI Advantage:** Compiles to a native C-compatible library natively, minimizing overhead across the FFI boundary.

### 2.3 Julia (The Idle Heavyweight)
* **Role:** Idle / Reserved for Future Expansion.
* **The "Sledgehammer vs. Scalpel" Strategy:** Julia is unparalleled for massive sparse matrices. However, a 9x9 grid constraint problem is a thumbtack, and Julia is a sledgehammer. The latency of crossing *two* FFI boundaries (Dart -> Rust -> Julia) defeats the purpose.
* **Action Plan:** Leave the Julia FFI bindings intact in the build scripts to avoid breaking CMake/Gradle configurations, but **do not invoke the Julia runtime**. In production, removing Julia entirely will be necessary to save 50MB-100MB of bundle size and reduce baseline memory footprint.
